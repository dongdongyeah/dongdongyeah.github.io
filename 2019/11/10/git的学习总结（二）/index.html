git的学习总结（二）

2019-11-10

三个区域

工作区：本地存放代码的地方
暂存区：暂时存放修改的地方
版本库：保存你的所有版本

两个命令

git add：将工作区内容添加到暂存区
git commit：将暂存区的内容提交到版本库

两个撤销

git checkout：将修改过但是还没添加到暂存区的东西给删除
git reset HEAD：将添加到暂存区，但是还没提交的版本库的东西给删除掉

模拟一个文件的使用

（1）本地创建版本库

$ mkdir fileName  //创建空目录（也叫空文件夹）
$ cd fileName  //进入目录
$ pwd  //显示当前目录
$ git init  //通过命令该目录变成git可管理的仓库

（2）提交文件

$ git add fileName  //注意！这里如果添加成功是没有消息反馈的
$ git commit -m "留下提示信息提示这是个什么操作"

（3）查看当前仓库状态

$ git status  //执行之后会反馈给你仓库相关信息，比如你当前的分支，比如哪些文件被修改过，但是还没提交

（4）查看修改前后的差异

上面说到执行 $git status 之后如果你对文件有所改动会提醒你，但是这个时候仅仅是提醒你文件改了，却没跟你说改啥了，这个时候我们可以用下面命令看看到底之前是改了什么内容。

$ git diff modifiedFileName

（5）提交修改

当我们确定了上面的修改信息确实是我们想保留下来的时候，可以重复上面(2)中的命令，就可以提交修改。

（6）查看提交记录

当你想看看自己提交过的所有记录，只有执行以下命令，就会帮你以从现在到从前的时间顺序罗列出你所有的提交记录。

$ git log

注意，上面的命令或详细罗列提交记录，一般一屏没法盛下这些记录，你可以按q键退出。
如果不想出现上面盛不下记录的情况，还可以使用一行记录的命令。

$ git log --pretty=oneline
