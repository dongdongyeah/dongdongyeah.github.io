git的学习总结（一）

2019-11-09

最近经常接触到git，今天有时间刚好整理一下我的有关git学习笔记。

认识git

git是什么?
分布式版本控制系统！
刚开始看到这个中文名字我是有点懵逼的，作为一个技术小白，这几个字我都认识，不知道怎么着拼在一起就有点晦涩难懂了。
在后来的学习中，才开始有了点自己的理解。下面是我自己的理解，或许不太正确，还请多多指教。
先挑着简单易懂的词，控制系统，控制什么呢，版本，什么版本呢？就是我们的代码，为什么我们的代码会有版本呢？当我们做一个项目的时候，肯定不是一气呵成的啦，得一步一步来，第一天干嘛，第二天干嘛，这是最基本的版本进阶，而git就是在记录着我们每一步脚印，它会帮我们记录下来每次的版本，不过不仅仅是记录，不然它就叫“版本记录系统”得了，更重要的作用是，它可以帮我们控制，当你发现我们第二天新增的东西不太好，向回到第一天的原始状态的时候，它又会帮我们回退到原来的版本，又或者这时候你又觉得其实第二天也不赖啦，真是个善变的女人呢，它还能帮你从第一天又穿回第二天的版本。不仅是如此的控制你的版本变动，据说它还可以帮忙控制团队协作，具体的我还没接触过，也没有很深刻的理解。分布式也是从团队这来的，对立着的是集中式的，就是一台服务器，每个队员往上提交东西，从上面要东西到自己电脑，再修改提交。而分布式，则是每个队员的电脑都是台服务器，有完整的内容，这样就不用联网向总服务器要东西了，而且这样也安全了许多，当东西丢了，每个人那都有完整的版本库，copy完了就解决了，如果是集中式，丢了就丢了。

git安装

首先在官网https://git-scm.com/安装git。
然后在git bash中自报家门。
如果安装成功，在桌面右击就可看到git bash here，单击进入即可。
在跳出的命令窗口中，输入你的用户名和邮箱。
$ git config --global user.name "Your Name"
$ git config --global user.email "email@example.com"
这里的参数 –global 表示本台机器的所有git仓库默认使用这个配置。
这样git的安装和基本配置就完成啦。
