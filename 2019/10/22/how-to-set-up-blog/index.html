记hexo+github博客搭建

2019-10-22 • 标签: 学习笔记

  一直想自己搭建个博客，但由于自身能力不足以及时间问题，总感觉这是个大工程，没勇气开始。
  今天看了个美美的博客，心痒痒的，刚好也有点空闲，便开始着手。以此文章记录一下。首先当然是安装下要用到的软件喽，顺便了解一下。

安装软件

1.Node.js

  node.js是一种运行环境，可以使javascript运行在服务器端。

2.Git

  git是个开源的分布式版本控制系统，可以有效、高速处理项目版本管理。提到了git，还得补充下，npm。
  npm是个包管理器。在安装node.js的时候会顺带帮你装上。怎么来的呢这个包管理器？一般我们想用到某个别人写好的JavaScript代码，都会先搜索、下载、解压，然后再使用，好麻烦呐。这时候，npm就派上用场了。大家都把自己的代码上传到npm官网，有需要直接npm下载，都不用各种找资源。

使用GitHub

甩个github链接先

1.注册

  注册这个不难，就是输个名字啊密码啊邮箱地址啊，不过哦，用户名谨慎取呀，伴你一生哈哈哈。

2.创建仓库

  这里提一句，仓库名字应该是username.github.io。我也不知道为什么反正大佬们都这么取。

3.配置SSH

  公钥和密钥这对钥匙用来打通本地git和线上github。

 (1)生成SSH keys

  这里，Git Bash上场啦。在命令框内输入 $ ssh-keygen -t rsa -C "邮箱地址"。然后系统让你输密码了，如果你没输的话，以后提交代码就省了输密码验证这一繁琐的环节了。这个设置是防止别人往你项目提交内容。不过，身为技术小白的我，完全不care这个问题，简单是我的唯一追求。然后就是一直enter键啊，直到出现了一个方形的框框，里面有一些个字母啊符号啊，当时没截图，现在也懒得折腾了。对了，在生成密钥的时候仔细看着会提示你已经把密钥存放在某某本地文件夹里。
  好啦，这个时候ssh keys就生成了。快去本地文件了看看吧。

 (2)添加SSH keys到GitHub

  在本地文件夹里找到id_rsa.pub文件，这个是公钥。然后来到github这儿，打开你的设置界面，有个SSH and GPG keys，在这把从id_rsa.pub文件里复制过来的东西在key里粘上，添加就ok啦。
  然后测试一下能不能通过SSH链接到github。在git bash里，输入以下代码
$ssh -T git@github.com,然后按照提示yes回车。这就都配置成功了。

4.搭建hexo博客

  首先使用npm命令安装hexo,输入命令：npm install -g hexo。然后在本地创建一个与仓库中博客项目同名的文件夹，使用git bash进入这个文件夹。使用以下命令
$hexo init
$npm install 安装依赖包
npm install hexo-deployer-git --save 确保git部署
$hexo g
$hexo s
然后就可以在浏览器总输入localhost:4000查看啦。

5.将博客部署到仓库

  首先在github个人主页的仓库中复制博客项目的SSH码，一定要是SSH码而不是HTTPS哦。然后呢，打开我们本地的博客项目文件中的_config.yml，把复制的SSH码加到“repository：”后面，冒号后是有空一格的。最后呢，执行下指令 $hexo g和$hexo d，这就部署成功啦！！！开心不，赶紧去浏览器输入你的博客地址看看呀。
